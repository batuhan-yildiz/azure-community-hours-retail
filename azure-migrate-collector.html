Discover Servers and Workloads Using Azure Migrate Collector

A fast and secure approach to discovering your IT estate without requiring direct Azure connectivity.

The Azure Migrate Collector is a discovery tool used to quickly identify servers and workloads across your IT estate without requiring direct Azure connectivity.

You deploy the collector on a Windows Server to scan VMware environments, as well as physical or virtual servers running on any hypervisor or public cloud.

The collector captures server configurations, performance data, installed software, SQL Server and PostgreSQL instances, and web applications such as .NET on IIS and Java on Tomcat.

Key Capabilities

Connectivity: Offline discovery
Time to discovery: Quick setup
Assessment types: Lift and shift, modernization
Performance-based sizing: Yes
Guest & workload discovery: Yes
Dependency mapping: No
Execute migrations: No

Azure Migrate Collector Deployment Overview

  • Create a new Azure Migrate project
  • Download the Azure Migrate Collector
  • Run the installer and configure the collector
  • Provide vCenter credentials
  • Provide guest and database credentials
  • Validate credentials
  • Start data collection
  • Review collected data and resolve any issues
  • Enable incremental data collection to retry failed workloads
  • Export collected data
  • Upload data to Azure Migrate
  • Enrich inventory with custom and reserved tags
  • Create reports, business cases, and assessments
  • Import additional inventory if required
